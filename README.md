## 文件夹分类规则

| 文件夹 | 用途 |
|--------|------|
| `document/` | 存放文档 |
| `save/` | 存放游戏存档 |
| `img/` | 存放游戏图片 |
| `date/` | 存放游戏数据 |

---

## 各文档职责定义

| 文件 | 职责 | 说明 |
|------|------|------|
| `document/策划案.md` | 游戏整体设计思路 | GDD 游戏设计文件 |
| `document/剧情.md` | 游戏剧情 | 故事情节、角色个人剧情、支线剧情 |
| `document/角色设计.md` | 角色设定 | 背景、性格、爱好、能力、立绘等 |
| `document/交互管理.md` | 可交互项信息 | 名称、跳转项 |
| `document/道具管理.md` | 道具信息 | 名称、功能、获取途径、价格（商店购买/剧情获取，无法交互的物品）|
| `document/事件管理.md` | 行动信息 | 名称、消耗或获取的资源（对应交互中的功能） |
| `document/设计参考.md` | **核心设计参考源** | 记录游戏各系统的详细设计思路，其他文档均以此为依据生成和补充 |
| `date/数据.json` | 数据信息 | 所有数值数据 |

## 工作流约定

- **文件修改备份**：修改前将文件复制到根目录 `备份/` 文件夹，命名为 `名称_修改日期`
- **README.md 记录**：每次回复时更新
- **策划案更新**：仅同步更新 document/下的交互管理.md、道具管理.md、事件管理.md
- **设计参考.md** 为权威参考源，其他文档均以此为依据生成和补充
