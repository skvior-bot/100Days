## 内层游戏设定（一个人的孤独之旅）

> 关联文件：`AloneLife.md`

### 定位
- 作为彩蛋级内容，玩法不需要太多
- 核心设计 TP 系统做好即可

### 玩法流程
1. 开头一小段 RPG（类似魔泉）
2. 主角被诅咒或获得未知力量 → 转变为回合制 RPG（类兰斯10）
3. 时间以 **TP（Time Point）** 为最小单元，仅玩家行动后时间才流逝
4. 流程：行动 → 看动画（1TP）→ 行动（重复）
5. 最终解除诅咒 → 世界恢复原状

### TP 系统
- 1TP 的时间长度以走路动画的实际时间为基准
- 对话动画时长取决于角色说话字数
- 其他行动均以此标准换算

## 游戏全流程

```
.exe启动 → 主界面（标题页）
  → 新的开始 → 初始剧情（背景+目标，可能漫画形式）→ 教程（鼠标点击交互）
  → 正式游戏（目标：把游戏做完）
  → 结局判定（构思中）
  → [达成真结局] → endPV（总结剧情CG + 参与人员名单，结尾显示打开制作的游戏）
  → 返回主界面，标题/背景改变（100Days → AloneLife）
  → 进入内层游戏（玩家开发完成的游戏）
  → 通关后双背景+双标题，可自由选择
  → 二周目（可能无新内容）
```

## UI 布局

- **标题**：上方正中心，巨大标题 logo
- **四个按钮**（下方从左到右排列）：
  - 新的开始：进入游戏，开始新的轮回
  - 继续游戏：点击弹出玩家存档列表，可任意选择
  - 设置：画面、文字效果、音量等设置，可能包含操作提示
  - 退出游戏：关闭程序
- **注**：之后用图片表达，文字不够形象

## 核心玩法循环

```
开发进度不够 → 需要提高效率
                 ↓
          购买更好设备 → 需要钱
                          ↓
                    赚钱（打工/直播/接广告等）
                          ↓
                    买设备 → 效率提高
                          ↓
                    开发游戏 → 进度增加
                          ↓
                进度达到了 → 解锁新剧情
```

## 角色与同伴系统

### 招募方式
- 剧情驱动、事件触发

### 角色能力维度
每个角色有不同的开发能力，按比例转化为对应项的开发进度：

| 能力 | 说明 |
|------|------|
| 程序 | 影响程序相关开发进度 |
| 美术 | 影响美术相关开发进度 |
| 文案 | 影响文案相关开发进度 |
| 音乐 | 影响音乐相关开发进度 |

### 互动方式
- 聊天 → 加深羁绊
- 约会 → 加深羁绊，解锁个人剧情
- 帮助解决个人事件 → 提高角色能力

### 羁绊奖励
- 解锁个人剧情
- 获得专属道具（提高开发效率/能力，玩家或角色）
- 角色能力提升（帮助解决对应事件）
