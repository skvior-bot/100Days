# UI 设计

> 职责：整理游戏 UI 的设计思路
> 参考依据：设计参考.md > 交互设计（交互树、事件结算器）

---

## 主界面（标题页）

- 标题位于上方正中心，巨大的标题 logo
- 四个按钮位于下方，按顺序从左到右排列：
  - **新的开始**：进入游戏，开始新的轮回
  - **继续游戏**：点击弹出玩家存档列表，可任意选择
  - **设置**：画面、文字效果、音量等设置，可能包含操作提示
  - **退出游戏**：关闭程序

## 游戏内界面

### 房间主场景

- 房间内展示可交互物品：电脑、床、书架、门等
- 点击交互物弹出功能列表方框
- 道具放置有视觉反馈：
  - 书本增多 → 书架变满
  - 食物放入冰箱
  - 加成类物品放入展示柜

### 电脑交互界面

**主选项层（4个按钮）：**
- 游戏开发 / 娱乐 / 工作 / 网购

**游戏开发子界面：**
- 四个方向按钮：程序·美术·音乐·文案（2×2排列）
- 每个按钮下方有完成度条（当前值/最大值）
- 最上方是总完成度条
- 选择方向后放入下方的结算器方框

**娱乐子界面：**
- 看动画：列表显示可选动画（一次5部，每周随机刷新），每个有进度条
- 玩游戏：列表显示已购买的游戏（需购买解锁），每个有进度条
- 看课程：按初级/中级/高级分类

**工作子界面：**
- 直播：选择直播游戏或杂谈
- 发视频：选择已通关游戏或已看完动画

**网购子界面：**
- 顶部三个类别标签：日常 / 娱乐 / 设备
- 下方显示当前类别下可购买的物品列表
- 可选择数量，可多选

### 事件结算器

- 位于交互弹出界面下方
- 多个小方框横向排列（太多会折叠）
- 被选择的事件依次放入方框中
- 点击方框中的事件可取消（清空当前方框，后方事件前移）
- 最右侧有结算按钮 → 点击依次结算所有事件
- 右上角小 X（退出）→ 清空方框，退出交互界面，无数据变化

## 地点切换界面

点击门后弹出列表，显示可前往地点（不包括当前所在地）
- 每个地点显示名称 + 消耗时间/体力
- 选择后触发移动事件

- 背景更换

## 视觉风格

- 黑白简约画风
- 参考风格：妹同居生活
- 2D 画面
- 待补充具体风格规范和配色方案

## UI 框架要点

- 交互 → 选择项目 → 放入结算器 → 结算 → 更新数据
- 不结算退出则不触发任何事件、不改变数据
